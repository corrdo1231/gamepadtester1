What is controller polling rate?

Polling rate describes how frequently new input states are observed over time. On this page, the result is based on browser-visible input updates, not a direct hardware bus reading.

Why browser polling rate may differ

Browsers sit above the operating system, driver, wireless stack, and controller firmware. That means the timing reported here is useful for comparison, but not a guaranteed hardware-level polling rate measurement.

How to interpret the result

Smaller intervals and steadier spacing usually feel more consistent. A wide spread between minimum and maximum intervals can point to browser scheduling limits, Bluetooth variability, or inconsistent input movement during the test.

Best way to test

Use a wired connection when possible, start the test, then move a stick in smooth circles or press buttons rhythmically until the timer ends.

Polling rate FAQ

Quick answers for understanding what this browser timing tool can and cannot tell you.

Is this the true hardware polling rate?

No. This page reports a browser-observed input update rate, which is shaped by the browser event loop and Gamepad API visibility.

Why does the page say no input detected?

That usually means the controller connected but did not produce enough changing input during the test window. Start again and keep a stick moving or press buttons repeatedly.

Why do Bluetooth and USB look different?

Wireless transport, operating system buffering, and browser timing can all change how often new input states reach the page.

What counts as an excellent result?

In this tool, excellent means frequent and fairly even browser-observed updates during active input. It is a stability hint, not a certification of the controller itself.
